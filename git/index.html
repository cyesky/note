                # 删除初始化项目

                git rm -rf .git

                # 设置默认推送地址

                git push --set-upstream origin master

                这样修改之后，就不用每次提交时输入 git push -u origin master ，直接使用git push即可。其中 -u 是 --set-upstream的缩写。

                # .gitignore文件

                # 文件不起作用

                .gitignore 只能忽略那些原来没有被 track 的文件，如果某些文件已经被纳入了版本管理中，则修改.gitignore 是无效的。解决方法就是先把本地缓存删除（改变成未 track 状态），然后再提交:

                git rm -r [文件/.] --cached .
git add .
git commit -m 'update .gitignore'

                # 忽略规则

                使用 Git 管理代码的过程中，可以修改 .gitignore 文件中的标示的方法来忽略开发者想忽略掉的文件或目录，如果没有 .gitignore 文件，可以自己手工创建。

                以斜杠“/”开头表示目录；
以星号“\*”通配多个字符；
以问号“?”通配单个字符
以方括号“\[\]”包含单个字符的匹配列表；
以叹号“!”表示不忽略(跟踪)匹配到的文件或目录。
注意： git 对于 .gitignore配置文件是按行从上到下进行规则匹配的

                # 删除分支

                在用 git 管理项目代码时，有时会创建不同名称的分支，以保证代码的合并和紧急回滚，bug 临时修复等。等项目开发完后，就需要执行一些删除分支操作，保证代码的完整性。

                1.列出本地分支：
git branch
2.删除本地分支：
git branch -D dev
git branch --delete dev
3.删除本地的远程分支：
git branch -r -D origin/dev
4.远程删除git服务器上的分支：
git push origin -d dev
git push origin --delete dev

                注意：git 命令区分大小写，例如-D 和-d 在不同的地方虽然都是删除的意思，并且它们的完整写法都是--delete，但简易写法用错大小写会执行失败。

                # 通过 https 提交 git，记住密码。

                凭据可以设置为全局，也可以设置为仅仅在当前的项目下起作用。

                如果想使用全局，则设置为： git config –global credential.helper store [–file=git_credentails]

                如果仅仅设置当前项目有效，则进入项目代码目录下，运行： git config credential.helper store [–file=.git_credentails]

                上面的 –file=.git-credentials 表示你的用户名密码存储的文件目录结构，即存放在哪里由你指定。 默认 .git_credentails 文件

                # 方法一（手动）

                1） 在你的用户目录下新建一个文本文件 .git-credentials

                用户目录:

                • windows:C:/Users/username

                • mac os x:/Users/username

                • linux:/home/username

                touch ~/.git-credentials

                vim ~/.git-credentials

                2）添加一行这个内容内容

                https://{username}:{passwd}@github.com

                上述{username}和{password}要换成你的 github 的账号名和密码，例如： https://admin@cyesky.com:mmmmm@github.com、那个平台写哪个后缀域名。其中 用户名，特殊字符要 url 转码，例如 @ 应写 %40

                3）在用户目录下的.gitconfig 文件最后添加以下代码

                [credential]
    helper = store

                或者执行命令git config --global credential.helper store ，上述命令会在~/.gitconfig文件末尾添加如上配置。

                亲测有效 经过上述三步配置之后, 你 push 代码到 github 时, 便无需再输入用户名密码了

                # git config命令的使用

                git config --global命令实际上在操作用户目录下的.gitconfig文件。我们 cat 一下此文件(cat .gitconfig), 其内容如下:

                [user]
    name = alice
    email = alice@aol.com
[credential]
    helper = store

                git config --global user.email "admin@cyesky.com"操作的就是上面的 email

                git config --global credential.helper store操作的就是上面最后一行的值
